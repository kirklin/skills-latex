# cn-doc — 中文通用文件 base 模板

中国用户日常「文件」的通用排版模板，样式为办公文档常见惯例：

- 大标题黑体二号居中（可选文号行，仿宋居中）；
- 中式标题层级 **一、**（黑体）／**（一）**（楷体）／**1.**（宋体加粗），
  标题与正文一样首行缩进两字；第四层「（1）」按惯例直接写在句中；
- 正文宋体小四、首行缩进两字、1.5 倍行距、Word 默认页边距；
- 全框线表格、中式表题「表 1　××」；
- 附件说明（左空两字）与右对齐落款（署名单位 + 成文日期）；
- 页码「— 1 —」居中。

示例内容是一份完整的虚构培训通知。通知、报告、总结、请示、函、制度、
方案说明等场景都从这一份底座改出，见下文「按场景改」。

## 文件结构

| 文件 | 作用 |
|---|---|
| `main.tex` | 唯一源文件 — 导言区（版面 / 层级样式）+ 示例正文 |

## 编译

```bash
bash scripts/compile.sh templates/cn-doc/main.tex --preview
```

`compile.sh` 检测到 ctex 自动选用 XeLaTeX；中文字体由 ctex 按操作系统自动选择，
macOS / Windows / Linux 开箱即用。

## 按场景改

| 场景 | 怎么改 |
|---|---|
| 通知（示例即是） | 换标题、主送单位、正文、落款即可 |
| 报告 / 总结 / 方案 | 删除文号行与主送单位；标题下加一行居中的「单位名称　2026 年 7 月」；落款按需保留 |
| 请示 / 函 | 保留主送单位；结束语改「特此请示，恳请批复。」「专此函达。」等 |
| 制度 / 办法 | 长文档取消 `\tableofcontents` 两行注释；条款式结构可把 `section` 的 `name` 改为 `{第,章}`、`number` 保持 `\chinese{section}`，`subsection` 改为「第 X 条」同理 |
| 会议纪要 | 标题改「××会议纪要」，文号行改为时间地点行，删除落款或改为记录人 |

## 常用调整

- **字体**：跨平台一致输出用 TeX Live 自带字体集：
  `\documentclass[zihao=-4, fontset=fandol]{ctexart}`。
- **字号**：`zihao=-4` 为小四；改 `zihao=5`（五号）更紧凑。
- **行距**：`\linespread{1.5}` 一行，按需改 1.3 或 2。
- **页码**：`fancyhdr` 段里改 `\fancyfoot`；不要页码则整段删除并去掉
  `\pagestyle{fancy}`。
- **插图**：已加载 `graphicx`，`\includegraphics[width=0.8\textwidth]{图片}` 即可。
- **公式 / 代码 / 参考文献**：按需在导言区补宏包 —— 公式加 `amsmath`，
  代码块加 `listings`，国标参考文献加 `gbt7714`（配 `.bib` 文件与
  `\bibliographystyle{gbt7714-numerical}`）。

## 边界说明

本模板取企业 / 机构通用样式（宋体小四），适合日常文件。需要严格遵循
GB/T 9704—2012 的党政机关公文（红头、发文字号、版记、仿宋三号正文）时，
请改用 `cn-gongwen` 模板 —— 它逐条落实该国标并附实测校验。
